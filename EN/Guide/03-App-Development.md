# 03. App Development

## Goal

Establish the default development path: build an external app first, then decide whether deeper changes are necessary.

## Core pages

- `https://developer.flipper.net/flipperzero/doxygen/applications.html`
- `https://developer.flipper.net/flipperzero/doxygen/app_manifests.html`
- `https://developer.flipper.net/flipperzero/doxygen/apps_on_sd_card.html`
- `https://developer.flipper.net/flipperzero/doxygen/app_examples.html`
- `https://developer.flipper.net/flipperzero/doxygen/app_publishing.html`

## Concepts you must understand

- `application.fam`
- `.fap`
- the difference between external apps and built-in firmware apps
- the difference between asset folders and data folders
- the pattern value of official examples

## Suggested order

1. `App Development`
2. `FAM`
3. `FAP`
4. `App Examples`
5. `Publishing`

## Expected outcome

- understand the basic structure of an external app
- understand the role of the manifest
- treat official examples as a pattern library
