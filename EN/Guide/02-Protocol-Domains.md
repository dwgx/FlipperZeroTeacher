# 02. Protocol Domains

## Goal

Organize knowledge by capability domain instead of by scattered use cases.

## Core pages

- `https://docs.flipper.net/nfc`
- `https://docs.flipper.net/sub-ghz`
- `https://docs.flipper.net/ibutton`
- `https://docs.flipper.net/zero/u2f`

## Main domains to keep separate

- NFC
- Sub-GHz
- RFID
- iButton
- Infrared
- BadUSB
- GPIO
- U2F

## Reading focus

- each domain has its own hardware boundary, menu entry, and file formats
- domains should not be mixed together
- device features, apps, scripts, and hardware expansion belong to different layers

## Expected outcome

- classify a request by protocol domain first
- know which official pages and file formats to inspect next
